| **UBS.UB: Incremental Build-Test Cycles - UBS.UB.UD: Principle Extraction Discipline** | Principle extraction is what converts build skill (tacit, person-bound, session-bound) into composable knowledge (explicit, transferable, usable across cycles and into OE System design). Extraction is the mechanism that links individual learning cycles → system design → the UDO. | The standing practice of writing exactly one design principle before the session closes or the next cycle opens — in a fixed format: 'Decision: [what decomposition variable was isolated] | Outcome: [binary result + observable change] | Rule: [what to apply in the next build].' IS a structured extraction act, not a note or journal entry: one traceable rule with a fixed schema. IS NOT optional or deferrable: the principle written immediately after the build is 10x more actionable than one written from memory the next day. IS NOT valid when scope creep occurred during the build: cycles where more than one variable changed cannot yield one clean traceable principle. | (1) Build cycle completes and A.C. passes. Rule: [what to do in the next build because of this]." (4) Commit principle to ILE. (5) Only then open the next cycle. Gate: principle is written in ILE within the same session as the build — not deferred to a future session. | The ELF micro-cycle (Learn → Build → Test → Validate → Measure) applied to each individual build cycle. ELF is not only the macro-level learning process — it is a micro-cycle applicable to each build. The "Measure" step is principle extraction. Without it, the ELF cycle is structurally incomplete, and learning does not compound across cycles. | ELF micro-cycle applied → each cycle has a Measure step → Measure = write one principle → principle stored in ILE → ILE accumulates principles across cycles → principles compound → mental model grows explicitly → subsequent A.C.s are better-designed → cycles produce sharper principles → compounding accelerates → Capability Gap closes at increasing speed. | P5 (enables UBS.UB.UD / amplifies UBS.UB): "Apply ELF as a micro-cycle to every build: each cycle must close with a written Measure step (one principle, decision-outcome-rule format) before the next cycle begins. This makes ELF recursive — each cycle improves the quality of the next. | ILE Learning Book (principle repository — cumulative across all cycles).
